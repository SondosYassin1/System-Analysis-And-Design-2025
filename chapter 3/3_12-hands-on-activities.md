# 3.12 Hands-On Activities

## Activity 1: Create Use Case Diagram (60 minutes)

### Scenario: Library Management System

**System Description:**
A library system for students and staff to borrow books, manage accounts, and handle reservations.

**Requirements:**
- Members can search and borrow books
- Members can reserve books that are checked out
- Librarians manage book inventory
- System sends overdue notices
- Members can renew books online
- Admin manages member accounts

**Your Tasks:**
1. Identify all actors (minimum 4)
2. Identify all use cases (minimum 8)
3. Create complete use case diagram
4. Include at least one <<include>> relationship
5. Include at least one <<extend>> relationship

**Deliverable:** Use case diagram in Lucidchart or Draw.io

---

## Activity 2: Write Detailed Use Case Specification (45 minutes)

### Scenario: Online Food Delivery

**Use Case:** Place Food Order

**Your Tasks:**
1. Write complete use case specification including:
   - Use case name and ID
   - Actors
   - Preconditions
   - Main success scenario (10-15 steps)
   - At least 3 alternative flows
   - Postconditions
   - Special requirements

**Template provided in section 3.6**

## Activity 3: User Story Writing Sprint (45 minutes)

### Scenario: School Management System - Attendance Module

**Requirements:**
Teacher needs to track student attendance for each class session.

**Your Tasks:**
1. Write 8-10 user stories for attendance tracking
2. Include acceptance criteria for each story
3. Estimate story points
4. Prioritize (Must/Should/Could)
5. Create story cards in format suitable for Jira

**User Roles to Consider:**
- Teacher
- Student
- Administrator
- Parent

---

## Activity 4: Story Mapping Workshop (60 minutes)

### Scenario: Student Assignment System

**Goal:** Map complete student assignment workflow

**Your Tasks:**
1. Identify 5-7 main activities (backbone)
2. List 3-5 user stories under each activity
3. Arrange stories by priority (top = high)
4. Draw MVP line
5. Define Release 1 and Release 2 scope

**Activities might include:**
- Create assignment
- Submit work
- Grade submission
- View feedback
- Track progress

**Deliverable:** Visual story map (can use Miro, sticky notes, or drawing tool)

## Activity 5: Format Conversion Exercise (30 minutes)

### Part A: Use Case → User Stories

**Given this use case:**

**UC-020: Student Withdraws from Course**

**Main Flow:**
1. Student logs in
2. Student views enrolled courses
3. Student selects course to withdraw
4. System checks withdrawal deadline
5. Student confirms withdrawal
6. System processes withdrawal
7. System calculates refund
8. System sends confirmation

**Your Task:** Convert to 4-6 user stories with acceptance criteria

### Part B: User Stories → Use Case

**Given these stories:**
- Teacher creates quiz
- Teacher adds questions
- Teacher sets time limit
- Teacher publishes quiz
- Student takes quiz

**Your Task:** Create one comprehensive use case with main flow and alternative flows

## Activity 6: Peer Review Exercise (45 minutes)

### Process

**Phase 1: Create (20 min)**
- Each student creates use case diagram for "Hotel Reservation System"
- Minimum 6 actors, 12 use cases

**Phase 2: Review (15 min)**
- Exchange diagrams with partner
- Use checklist to review:
  - [ ] Actors outside boundary?
  - [ ] Use case names are verb phrases?
  - [ ] Appropriate granularity?
  - [ ] Relationships used correctly?
  - [ ] Diagram is readable?

**Phase 3: Discuss (10 min)**
- Discuss findings
- Suggest improvements
- Learn from each other

## Activity 7: Tool Practice - Lucidchart (45 minutes)

### Tasks

1. **Setup (5 min)**
   - Sign up for Lucidchart education account
   - Create new document
   - Select UML use case template

2. **Create Diagram (25 min)**
   - Model "Restaurant Ordering System"
   - Include: Customer, Waiter, Kitchen, Payment System
   - Use cases: Order food, modify order, prepare dish, process payment, generate receipt
   - Add relationships

3. **Enhance (10 min)**
   - Add colors and styles
   - Group related use cases
   - Add notes/annotations
   - Export as PNG

4. **Share (5 min)**
   - Generate shareable link
   - Add to portfolio

## Activity 8: Group Project - School System Module (90 minutes)

### Scenario

Your team is designing the "Parent Portal" module for the School Management System.

### Team Roles (4 people)
- **Business Analyst:** Gathers requirements
- **Use Case Designer:** Creates diagrams
- **Story Writer:** Writes user stories
- **Tester:** Defines acceptance criteria

### Deliverables

1. **Use Case Diagram** showing all parent interactions
2. **3 Detailed Use Case Specifications**
3. **20 User Stories** organized as an epic
4. **Story Map** for parent portal
5. **Presentation** (10 min) to class

### Features to Include
- View student grades
- Communicate with teachers
- Update contact information
- Make payments
- View attendance
- Sign permission slips
- Schedule meetings

## Activity 10: Final Integration Exercise (120 minutes)

### Project: Complete School Management System

**Your Task:** Create complete use case model

### Deliverables

1. **Actor List** with descriptions
2. **Use Case List** (30+ use cases)
3. **5 Use Case Diagrams** (one per major subsystem):
   - Student Academic Module
   - Teacher Course Management
   - Administrative Functions
   - Parent Portal
   - System Integration

4. **10 Detailed Use Case Specifications**
5. **Epic Structure** with 50+ user stories
6. **Story Map** for complete system
7. **Traceability Matrix** (requirements → use cases → stories)

### Quality Checklist
- [ ] All actors identified and documented
- [ ] Use cases complete and non-overlapping
- [ ] Diagrams clear and professional
- [ ] Specifications follow template
- [ ] Stories have acceptance criteria
- [ ] Appropriate relationships used
- [ ] Consistent naming conventions
- [ ] Proper granularity throughout
